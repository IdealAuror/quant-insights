Strategy Overview

A mean-reversion asset rotation strategy across four low-correlation ETFs. When an asset's short-term return significantly exceeds its long-term quarterly average, the strategy underweights it; when it underperforms, the strategy overweights it.

Rebalanced weekly. No market timing. No stock selection. Pure asset allocation driven by a single deviation formula.

Why Base Weight Doesn't Matter

When no asset hits the 15% floor, the ratio between any two assets is:

Finali / Finalk = (B - di) / (B - dk)

Changing B shifts both sides equally — only relative differences between deviations matter.

Mathematical Analysis: Why This Strategy Works

1. Statistical Foundation of Mean Reversion

Mean reversion is one of the most robust empirical findings in finance. Its theoretical basis traces to the Ornstein-Uhlenbeck process:

// OU process (continuous-time mean reversion model)
dXt = θ(μ - Xt)dt + σdWt

// θ: speed of reversion   μ: long-term equilibrium   σ: volatility

When price deviates from equilibrium μ, the drift term pulls it back. The half-life measures reversion speed:

Half-life = ln(2) / θ ≈ 63 trading days (~1 quarter)

// The short window is set to exactly 63 days — aligned with the typical half-life
// This captures the optimal point: deviation has fully materialized and is about to revert

Academic support: Poterba & Summers (1988), Fama & French (1988) both find significant mean reversion in stock prices over 1-4 year horizons.

2. Why the Deviation Metric Captures Reversion

The deviation R_short - R_long / 5 measures: how much the current quarter deviates from the long-term quarterly average.

// Decomposing long-term return into 5 quarterly components
R_long = (1+r1)(1+r2)(1+r3)(1+r4)(1+r5) - 1

// R_long / 5 approximates "average quarterly return"
// When R_short >> R_long/5: current quarter abnormally strong → revert down
// When R_short << R_long/5: current quarter abnormally weak → revert up

Key mathematical properties:

  • Self-normalizing: Eliminates absolute return differences across assets (gold 8%/yr vs bonds 3%/yr), only compares relative deviation
  • Symmetric: Deviation > 0 and < 0 treated identically — no directional bias
  • Dimensionless: Returns are unitless, so deviation can be compared across any asset class

Conclusion: The strategy's effectiveness does not depend on precise parameter choice. As long as the window stays within "approximately one quarter" (40-80 days), deviation signals align. This is robustness by design, not overfitting.

6. Theoretical Limitations

Mean reversion assumes: prices oscillate around equilibrium with no persistent trend. When this breaks down:

// If θ in the OU process approaches 0 (no reversion force):
dXt = σdWt   // degenerates to random walk — mean reversion fails

// This happens in:
// - Structural bull markets (e.g., 2023 AI-driven Nasdaq rally)
// - Monetary regime shifts (e.g., 2020 Fed unlimited QE)
// - Geopolitical shocks (e.g., 2022 Russia-Ukraine gold spike)

Mitigation: A trend filter (e.g., 200-day MA) could pause rebalancing during strong trends, but adds complexity — conflicting with the "one formula" simplicity.

Strengths & Weaknesses

Strengths

  • All-weather: A-shares, US equities, gold, bonds — at least one asset shines in any cycle
  • Excellent hold experience: Max DD under 10%, easy to stick with
  • Transparent: One formula, no black box
  • High capacity: All four ETFs trade billions daily

Weaknesses

  • Trend FOMO: In sustained rallies, keeps reducing the winner and underperforms buy-and-hold
  • Mean-reversion failure: In strong trends, deviation signal fires wrong repeatedly
  • No vol targeting: Ignores risk — volatile assets could be over-concentrated

# ETF Mean-Reversion Rotation Strategy
# Universe: 512890 / 513100 / 518880 / 511260

import jqdata
import pandas as pd
import numpy as np

CONFIG = {
    'short_window': 80,
    'long_window':  315,
    'base_weight':  0.40,
    'min_weight':   0.15,
}

def initialize(context):
    g.etfs = ['512890.XSHG', '513100.XSHG', '518880.XSHG', '511260.XSHG']
    g.short_window = CONFIG['short_window']
    g.long_window  = CONFIG['long_window']
    g.base_weight  = CONFIG['base_weight']
    g.min_weight   = CONFIG['min_weight']
    set_benchmark('000300.XSHG')
    set_option('use_real_price', True)
    set_order_cost(OrderCost(
        open_tax=0, close_tax=0,
        open_commission=0.0003, close_commission=0.0003,
        close_today_commission=0, min_commission=5
    ), type='fund')
    run_daily(weekly_rebalance, time='09:45')

def weekly_rebalance(context):
    prices = {}
    for etf in g.etfs:
        df = attribute_history(etf, g.long_window + g.short_window + 5,
                               '1d', ['close'], skip_paused=True)
        prices[etf] = df['close'] if df is not None and len(df) >= g.long_window else None

    raw_weights, total_raw = {}, 0
    for etf in g.etfs:
        if prices[etf] is None:
            raw_weights[etf] = g.min_weight
        else:
            c = prices[etf]
            short_ret = c.iloc[-1] / c.iloc[-g.short_window] - 1
            long_ret  = c.iloc[-1] / c.iloc[-g.long_window]  - 1
            deviation = short_ret - long_ret / 5
            raw_weights[etf] = max(g.base_weight - deviation, g.min_weight)
        total_raw += raw_weights[etf]

    for etf in g.etfs:
        w = raw_weights[etf] / total_raw
        order_target_value(etf, context.portfolio.total_value * w)
